## 1. 策略概述

### 1.1 核心原则

```yaml
Manus式饥饿营销:
  1. 极度稀缺: 每天只放出少量邀请码
  2. 申请门槛: 不是注册就能用，需要申请+审核
  3. 社交货币: 有邀请码 = 有身份
  4. 话题制造: "Thinkus邀请码"成为热门话题

目标效果:
  - 邀请码成为社交货币
  - 零成本获得病毒传播
  - 早期用户高质量
  - 建立产品稀缺感
```

```yaml
释放策略:

  目标: 筛选最优质的早期用户
  
阶段2 - 早期增长 (第3-8周):
  每日释放: 100-200个
  审核: AI初筛 + 人工复核
  目标: 建立口碑，保持稀缺
  
阶段3 - 扩大规模 (第9周+):
  每日释放: 500-1000个
  审核: AI审核为主
  目标: 加速增长，但仍保持门槛

永远不会:
  - 无限量开放注册
  - 取消申请流程
  - 让邀请码变得不值钱
```

## 3. 老用户邀请码

### 3.1 获取条件

```yaml
老用户如何获得邀请码:

规则: 不是想要就有，需要达成条件

条件1 - 付费用户:
  种子项目用户 (¥99/月): 0个邀请码/月
  成长项目用户 (¥299/月): 1个邀请码/月
  规模项目用户 (¥699/月): 2个邀请码/月
  企业项目用户 (¥1499/月): 3个邀请码/月
  
条件2 - 贡献者:
  提交有效反馈: +1个邀请码
  发现重大Bug: +2个邀请码
  社区优质内容: +1个邀请码

限制:
  - 每个用户最多持有3个未使用邀请码
  - 邀请码30天内未使用自动回收
  - 不允许买卖邀请码 (违规封号)
```

### 3.2 邀请码等级

```yaml
普通邀请码 (Common):
  获取: 付费用户每月配额
  有效期: 14天
  被邀请人福利: 跳过排队，直接注册
  数量: 有限

稀有邀请码 (Rare):
  获取: 成功邀请3人后解锁 / 重大贡献奖励
  有效期: 30天
  被邀请人福利: 跳过排队 + 7天成长项目体验
  数量: 极少

传说邀请码 (Legendary):
  获取: 特别贡献 / 官方活动
  有效期: 60天
  被邀请人福利: 跳过排队 + 30天规模项目体验
  数量: 稀有
```

### 3.3 邀请追踪

```yaml
邀请者可以看到:
  - 我的邀请码列表
  - 每个邀请码的状态 (未使用/已使用/已过期)
  - 谁使用了我的邀请码
  - 被邀请人是否付费 (脱敏信息)

邀请激励:
  - 被邀请人首次付费: 邀请者获得积分/折扣
  - 累计邀请排行榜 (月度/总榜)
  - 特别活动期间邀请加倍
```

## 4. 技术实现

### 4.1 排队系统

```typescript
// 申请排队
async function applyForWaitlist(input: {
  email: string
  projectIdea: string
  role: string
  referralSource?: string
  socialLinks?: string[]
}) {
  // 计算排名位置
  const position = await db.waitlist.count({ status: 'pending' }) + 1
  
  // 计算申请质量分数 (AI评估)
  const score = await evaluateApplicationQuality(input)
  
  // 创建申请
  const application = await db.waitlist.create({
    email: input.email,
    application: input,
    queue: {
      position,
      appliedAt: new Date(),
      score,
      priority: score > 80 ? 'high' : 'normal'
    },
    status: 'pending'
  })
  
  return {
    position,
    estimatedWait: calculateEstimatedWait(position)
  }
}

// 每日释放Job (凌晨2点执行)
async function runDailyRelease() {
  const config = await getDailyReleaseConfig()
  const toRelease = config.quota.remaining
  
  if (toRelease <= 0) return
  
  // 按分数排序，取前N个
  const applications = await db.waitlist.findMany({
    where: { status: 'pending' },
    orderBy: [
      { 'queue.priority': 'desc' },
      { 'queue.score': 'desc' },
      { 'queue.appliedAt': 'asc' }
    ],
    take: toRelease
  })
  
  for (const app of applications) {
    // 生成邀请码
    const code = generateInvitationCode()
    
    // 更新申请状态
    await db.waitlist.updateOne({
      where: { id: app.id },
      data: {
        status: 'approved',
        invitationCode: code,
        invitationExpiresAt: addDays(new Date(), 7)
      }
    })
    
    // 创建邀请码记录
    await db.invitationCodes.create({
      code,
      type: 'waitlist',
      boundTo: app.email,
      status: 'active',
      expiresAt: addDays(new Date(), 7),
      benefits: { skipWaitlist: true }
    })
    
    // 发送邮件
    await sendInvitationEmail(app.email, code)
  }
}
```

### 4.2 邀请码验证

```typescript
// 注册时验证邀请码
async function validateInvitationCode(code: string, email: string) {
  const invitation = await db.invitationCodes.findOne({ code })
  
  if (!invitation) {
    return { valid: false, reason: '邀请码不存在' }
  }
  
  if (invitation.status !== 'active') {
    return { valid: false, reason: '邀请码已使用或已过期' }
  }
  
  if (invitation.expiresAt < new Date()) {
    return { valid: false, reason: '邀请码已过期' }
  }
  
  // 如果是waitlist类型，检查是否绑定到该邮箱
  if (invitation.type === 'waitlist' && invitation.boundTo !== email) {
    return { valid: false, reason: '该邀请码已绑定其他邮箱' }
  }
  
  return { 
    valid: true, 
    benefits: invitation.benefits,
    tier: invitation.tier
  }
}

// 使用邀请码
async function useInvitationCode(code: string, userId: string) {
  await db.invitationCodes.updateOne({
    where: { code },
    data: {
      status: 'used',
      usedBy: userId,
      usedAt: new Date()
    }
  })
  
  // 如果是用户邀请码，记录邀请关系
  const invitation = await db.invitationCodes.findOne({ code })
  if (invitation.type === 'user_invite') {
    await recordInvitation(invitation.createdBy, userId)
  }
}
```

## 5. 营销节奏

### 5.1 发布节奏

```yaml
Week 1-2 (造势期):
  - 每日50个邀请码
  - 创始人社交媒体预热
  - KOL种子用户邀请
  - 目标: 积累3000+排队

Week 3-4 (引爆期):
  - 保持每日50-100个
  - 发布"排队破万"新闻
  - 用户晒邀请码截图
  - 目标: 排队破10000

Week 5-8 (持续热度):
  - 逐步增加到200/天
  - 用户案例分享
  - 保持稀缺感

Week 9+ (规模化):
```
