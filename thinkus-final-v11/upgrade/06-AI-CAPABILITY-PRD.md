# Thinkus AI能力增强方案 - 产品需求文档

> **版本**: v1.0 | **日期**: 2026-01-15
>
> **核心升级**: Skills技能系统 + Subagents并行协作 + MCP服务连接 + Browser自动化
>
> **目标**: 让AI高管更专业、更高效、更强大

---

## 一、背景与目标

### 1.1 当前系统能力

```yaml
已有能力:
  ✅ 18个AI高管，专业分工
  ✅ 专家讨论系统
  ✅ 用户专属实例
  ✅ 记忆系统

存在局限:
  ❌ AI能力靠Prompt硬编码，难以扩展
  ❌ 高管串行工作，效率受限
  ❌ 无法连接用户的外部服务
  ❌ 无法自动化操作网页
```

### 1.2 升级目标

```yaml
本次升级引入四大能力:

1. Skills (技能系统)
   - AI高管的能力模块化、可复用
   - 支持系统预设 + 用户自定义
   - 经验库升级为技能库

2. Subagents (子代理)
   - 复杂任务自动分解
   - 多个AI并行工作
   - 效率提升50%+

3. MCP (服务连接)
   - 连接用户的GitHub/Vercel/Stripe等
   - AI直接操作用户的服务
   - 实现真正的端到端交付

4. Browser (浏览器自动化)
   - 实时预览开发中的产品
   - 自动化测试
   - 竞品网站分析
```

### 1.3 用户价值

| 升级项 | 用户价值 |
|--------|----------|
| Skills | 生成的代码更专业、更规范、复用率更高 |
| Subagents | 开发更快，多人同时干活 |
| MCP | 代码直接推到我的GitHub，一键部署到Vercel |
| Browser | 看到AI实时开发界面，自动帮我测试 |

## 二、Skills 技能系统

### 2.1 概念定义

```yaml
什么是Skill:
  一个Skill = 知识 + 流程 + 模板 + 最佳实践
  
  类比: 员工的专业技能证书
  
  例如 "Next.js开发" Skill包含:
    - 知识: App Router原理、Server Components
    - 流程: 如何创建页面、如何做数据获取
    - 模板: 标准的页面结构、API路由模板
    - 最佳实践: 性能优化、SEO处理
```

### 2.3 Skill与AI高管的关系

```yaml
每个AI高管有默认技能 + 可加载额外技能:

Mike (CEO):
  默认: [需求分析, 项目规划, 商业模式]
  可加载: [saas-metrics, growth-hack, pricing]

David (CTO):
  默认: [技术选型, 架构设计, 代码审查]
  可加载: [nextjs, react-native, mongodb, stripe-integration]

Elena (设计总监):
  默认: [UI设计, 用户体验, 设计系统]
  可加载: [ui-minimal, ui-corporate, responsive-layout]

Frank (CFO):
  默认: [成本分析, 定价策略, 财务规划]
  可加载: [subscription, pricing, saas-metrics]
```

### 2.4 Skill加载时机

```yaml
自动加载:
  - 根据项目类型加载对应产品Skill
  - 根据技术栈加载对应技术Skill
  - 根据用户偏好加载用户Skill

示例:
  用户: "做一个SaaS订阅平台，用Next.js"
  
  自动加载:
  ├── 产品Skill: saas (多租户、订阅、权限)
  ├── 技术Skill: nextjs-app-router
  ├── 技术Skill: stripe-integration (订阅支付)
  └── 用户Skill: user/coding-standard (如果有)
```

### 2.5 Skill内容结构

```yaml
示例: ecommerce (电商平台) Skill

name: ecommerce
version: "1.0"
description: 电商平台开发最佳实践

knowledge:
  - 购物车应该存在客户端还是服务端？答：混合模式，未登录存localStorage，登录后同步到服务端
  - 库存扣减时机？答：下单时预扣，支付成功后确认，超时释放
  - 订单状态机：待支付→已支付→已发货→已完成/已取消/已退款

procedures:
  - name: 实现购物车
    steps:
      1. 创建Cart Context管理状态
      2. 实现addItem/removeItem/updateQuantity
      3. 计算小计和总价（考虑折扣）
      4. 未登录用户存localStorage
      5. 登录时同步到服务端

  - name: 实现支付流程
    steps:
      1. 创建订单，状态=待支付
      2. 调用Stripe创建PaymentIntent
      3. 前端使用Stripe Elements
      4. Webhook接收支付结果
      5. 更新订单状态

templates:
  - name: 商品列表页
    path: /templates/ecommerce/product-list.tsx
  - name: 购物车组件
    path: /templates/ecommerce/cart.tsx
  - name: 结算页
    path: /templates/ecommerce/checkout.tsx

best_practices:
  - 使用乐观更新提升购物车操作体验
  - 价格计算统一在服务端，防止篡改
  - Webhook处理要幂等
  - 敏感操作记录审计日志
```

## 三、Subagents 并行协作

### 3.1 概念定义

```yaml
什么是Subagent:
  主Agent将复杂任务分解，派生多个子Agent并行处理
  
  类比: 项目经理分配任务给团队成员，大家同时干活
  
  当前问题:
    用户提需求 → Mike分析 → David评估 → Elena设计 → 串行执行
    耗时: 3分钟 + 3分钟 + 3分钟 = 9分钟
  
  升级后:
    用户提需求 → Mike分解 → [David, Elena, Frank] 并行 → 汇总
    耗时: 1分钟 + 3分钟 + 1分钟 = 5分钟 (节省44%)
```

## 四、MCP 服务连接

### 4.1 概念定义

```yaml
什么是MCP (Model Context Protocol):
  让AI能够连接和操作外部服务的标准协议
  
  类比: 公司的供应商/合作伙伴网络
  
  价值:
    - AI可以直接推代码到用户的GitHub
    - AI可以直接部署到用户的Vercel
    - AI可以读取用户的Stripe收入数据
    - AI可以获取用户的Google Analytics数据
```
